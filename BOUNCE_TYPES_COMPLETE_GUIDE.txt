🎯 BOUNCE TYPE #1: REGULAR EMAIL BOUNCE
═══════════════════════════════════════════════════════════════════════════════

Error Message:
  "Address not found"
  "Your message wasn't delivered to info@flagsmith.com because the 
   address couldn't be found or is unable to receive email."

What It Means:
  ❌ The mailbox doesn't exist
  ❌ The email address is invalid
  ❌ The domain rejected the address
  ❌ The mailbox was deleted

Root Causes:
  • Typo in email address
  • Wrong email pattern (info@ instead of support@)
  • Mailbox doesn't exist on server
  • Company changed contact email
  • Mailbox is inactive/deleted

How to Identify:
  ✓ Error mentions "address not found"
  ✓ Error mentions "user doesn't exist"
  ✓ Error mentions "mailbox doesn't exist"
  ✓ SMTP code: 550, 551, 552, 553

Action to Take:
  ❌ REMOVE from email list immediately
  ❌ Do NOT retry
  ❌ Do NOT send again
  ✅ Mark as invalid
  ✅ Try alternative email pattern (support@, hello@, etc.)

Expected Bounce Rate:
  >90% (will bounce every time)

Example:
  Email: info@flagsmith.com
  Status: INVALID - Remove from list
  Alternative: Try support@flagsmith.com


🎯 BOUNCE TYPE #2: GOOGLE GROUP BOUNCE
═══════════════════════════════════════════════════════════════════════════════

Error Message:
  "The group you tried to contact (contact) may not exist, or you may not 
   have permission to post messages to the group."

What It Means:
  ⚠️  Email is a Google Group (mailing list), not a regular mailbox
  ⚠️  You don't have permission to post
  ⚠️  Group is private or restricted
  ⚠️  Group requires membership

Root Causes:
  • Domain uses Google Workspace
  • Email is configured as Google Group
  • Group is private/restricted
  • You're not a member of the group
  • Group doesn't allow external posts

How to Identify:
  ✓ Error mentions "group"
  ✓ Error mentions "permission to post"
  ✓ Error mentions "join the group"
  ✓ Sender is mailer-daemon@googlemail.com
  ✓ Domain uses Google Workspace
  ✓ Email pattern: contact@, info@, etc.

Action to Take:
  ⚠️  DO NOT send again (will bounce)
  ✅ Try alternative email pattern (support@, hello@, etc.)
  ✅ Check company website for contact form
  ✅ Look for LinkedIn company page
  ✅ Join the group if relevant
  ✅ Use different outreach method

Expected Bounce Rate:
  20-50% (may work if you join group)

Example:
  Email: contact@eightx.co
  Status: GOOGLE GROUP - Try alternative
  Alternative: Try support@eightx.co or contact form


🎯 BOUNCE TYPE #3: SOFT BOUNCE (TEMPORARY)
═══════════════════════════════════════════════════════════════════════════════

Error Message:
  "Mailbox full"
  "Service temporarily unavailable"
  "Try again later"
  "Connection timeout"

What It Means:
  ⏱️  Temporary issue, not permanent
  ⏱️  Mailbox exists but can't receive right now
  ⏱️  Server is temporarily down
  ⏱️  Rate limit exceeded

Root Causes:
  • Mailbox is full
  • Server is temporarily unavailable
  • Message is too large
  • Rate limit exceeded
  • Network issue

How to Identify:
  ✓ Error mentions "try again"
  ✓ Error mentions "temporarily"
  ✓ SMTP code: 421, 450, 451, 452
  ✓ Error is temporary, not permanent

Action to Take:
  ✅ RETRY after 24 hours
  ✅ Try up to 3 times
  ✅ Then remove if still failing
  ✅ Monitor bounce rate

Expected Bounce Rate:
  <5% (usually resolves)

Example:
  Email: support@company.com
  Status: SOFT BOUNCE - Retry tomorrow
  Action: Resend after 24 hours


🎯 BOUNCE TYPE #4: SPAM/AUTHENTICATION BOUNCE
═══════════════════════════════════════════════════════════════════════════════

Error Message:
  "Failed authentication"
  "SPF check failed"
  "DKIM signature invalid"
  "Message marked as spam"

What It Means:
  🚫 Your email failed authentication checks
  🚫 Your sender reputation is poor
  🚫 Your domain is not properly configured
  🚫 Your IP is blacklisted

Root Causes:
  • Missing SPF record
  • Missing DKIM signature
  • Missing DMARC policy
  • Sender IP is blacklisted
  • Poor sender reputation
  • Sending too many emails too fast

How to Identify:
  ✓ Error mentions "authentication"
  ✓ Error mentions "SPF", "DKIM", "DMARC"
  ✓ Error mentions "spam"
  ✓ Error mentions "blacklist"

Action to Take:
  ✅ Set up SPF record
  ✅ Set up DKIM signature
  ✅ Set up DMARC policy
  ✅ Check sender reputation
  ✅ Warm up IP gradually
  ✅ Reduce sending volume

Expected Bounce Rate:
  Variable (depends on configuration)

Example:
  Email: support@company.com
  Status: AUTHENTICATION FAILED
  Action: Fix SPF/DKIM/DMARC records
